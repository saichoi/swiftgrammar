1. Compile
사람이 이해할 수 있는 텍스트를 컴퓨터가 이해할 수 있도록 바이너리 코드(0과 1로 구성)로 바꾸는 것.
오류가 없는 경우에만 바이너리 코드를 생성.

1-1. 오류의 종류
Warning (경고) : 프로그램 실행에는 문제가 없지만 논리적인 문제가 생길 수 있다.(노란색 삼각형)
Error (오류) : 컴파일 자체를 할 수 없는 오류로 반드시 수정해야한다.(빨간색 원)

2. Link
프레임워크, 라이브러리를 연결해주는 과정.
컴파일과 링크를 실행 : command + R 버튼을 클릭하면 알아서 해준다.(IDE)

Build : compile -> link -> 실행파일 생성

2-1. 실행파일 생성
Debug Mode : 개발 과정에서 오류를 쉽게 찾을 수 있다.(디버그 정보 포함)
Release Mode : 최적화. 실행속도 빠르다.(디버그 정보 포함x)

3. Run
실행 파일 생성하기까지를 Compile Time
실행하는 시점을 Runtime
